# Refer & Earn — Complete Implementation Plan

**Platform**: Dofurs Pet Services Marketplace
**Feature**: Refer & Earn — ₹500 subscription credits for referrer + referee
**Status**: Ready to implement (all gaps identified, zero assumptions)

---

## 1. Feature Summary

| Aspect | Detail |
|---|---|
| Referrer reward | ₹500 Dofurs Credit added immediately on referee's successful sign-up |
| Referee reward | ₹500 Dofurs Credit added immediately on their own sign-up with a valid code |
| Credit type | Monetary credit wallet (INR balance), **not** tied to a specific service type — works for all services |
| Referral code | 9-char code: `DOF` + 6 uppercase alphanumeric (e.g. `DOFR4X9K2`) |
| Code generation | Auto-generated PostgreSQL trigger on every new `users` row insert |
| Self-referral | Blocked — code owner cannot redeem their own code |
| Re-use | Each user can only enter a referral code once (at sign-up only) |
| Credit usage | Applied as a discount at checkout in the booking flow (`ReviewConfirmStep`) |
| Shareable link | `https://dofurs.com/auth/sign-in?mode=signup&ref=DOFR4X9K2` |

---

## 2. Architecture Decision: Monetary Credit Wallet

The existing subscription credit system (`user_service_credits`) works as **per-service-type booking counts** tied to paid subscription plans. It cannot natively represent "₹500 applicable to any service."

**Decision**: Introduce a separate **Dofurs Credit Wallet** — a monetary (INR) balance per user — independent of the subscription system. This is cleaner, more flexible, and directly satisfies "applicable for all available services."

New concept at checkout: users can apply wallet credits as a **partial or full discount** on any booking, on top of or instead of existing subscription credits.

```sql
-- ============================================================
-- TABLE: referral_codes
-- One row per user, unique code, auto-generated by trigger.
-- ============================================================
CREATE TABLE IF NOT EXISTS referral_codes (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         uuid NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
  code            text NOT NULL UNIQUE,
  total_referrals int  NOT NULL DEFAULT 0,
  created_at      timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_referral_codes_code ON referral_codes(code);
CREATE INDEX IF NOT EXISTS idx_referral_codes_user_id ON referral_codes(user_id);

-- ============================================================
-- TABLE: referral_redemptions
-- One row per successful referral signup event.
-- ============================================================
CREATE TABLE IF NOT EXISTS referral_redemptions (
  id                      uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  referral_code           text        NOT NULL,
  referrer_user_id        uuid        NOT NULL REFERENCES users(id),
  referee_user_id         uuid        NOT NULL REFERENCES users(id),
  status                  text        NOT NULL DEFAULT 'pending'
                            CHECK (status IN ('pending', 'credited', 'failed')),
  referrer_credited_at    timestamptz,
  referee_credited_at     timestamptz,
  created_at              timestamptz NOT NULL DEFAULT now(),
  UNIQUE (referee_user_id)   -- each user can only be a referee once
);

CREATE INDEX IF NOT EXISTS idx_referral_redemptions_referrer ON referral_redemptions(referrer_user_id);
CREATE INDEX IF NOT EXISTS idx_referral_redemptions_referee ON referral_redemptions(referee_user_id);
CREATE INDEX IF NOT EXISTS idx_referral_redemptions_code ON referral_redemptions(referral_code);

-- ============================================================
-- TABLE: user_credit_balance
-- Monetary credit wallet per user (in whole rupees).
-- ============================================================
CREATE TABLE IF NOT EXISTS user_credit_balance (
  id                    uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id               uuid NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
  available_inr         int  NOT NULL DEFAULT 0 CHECK (available_inr >= 0),
  lifetime_earned_inr   int  NOT NULL DEFAULT 0,
  lifetime_used_inr     int  NOT NULL DEFAULT 0,
  created_at            timestamptz NOT NULL DEFAULT now(),
  updated_at            timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_user_credit_balance_user_id ON user_credit_balance(user_id);

-- ============================================================
-- TABLE: credit_wallet_transactions
-- Immutable audit log of every credit change.
-- ============================================================
CREATE TABLE IF NOT EXISTS credit_wallet_transactions (
  id               uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          uuid NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  amount_inr       int  NOT NULL,  -- positive = earned, negative = used/expired
  transaction_type text NOT NULL
                     CHECK (transaction_type IN (
                       'referral_reward_referrer',
                       'referral_reward_referee',
                       'booking_applied',
                       'booking_refunded',
                       'admin_grant',
                       'admin_deduct',
                       'expired'
                     )),
  reference_id     uuid,       -- referral_redemption.id or booking.id
  notes            text,
  balance_after    int NOT NULL,
  created_at       timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_cwt_user_id ON credit_wallet_transactions(user_id);
CREATE INDEX IF NOT EXISTS idx_cwt_type ON credit_wallet_transactions(transaction_type);

-- ============================================================
-- FUNCTION: generate_referral_code()
-- Generates a DOF + 6-char code avoiding confusable chars.
-- ============================================================
CREATE OR REPLACE FUNCTION generate_referral_code()
RETURNS text AS $$
DECLARE
  chars text := 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
  code  text := 'DOF';
  i     int;
BEGIN
  FOR i IN 1..6 LOOP
    code := code || substr(chars, floor(random() * length(chars) + 1)::int, 1);
  END LOOP;
  RETURN code;
END;
$$ LANGUAGE plpgsql;

-- ============================================================
-- FUNCTION + TRIGGER: auto-create referral_code on user insert
-- ============================================================
CREATE OR REPLACE FUNCTION trg_fn_create_referral_code()
RETURNS trigger AS $$
DECLARE
  new_code text;
  attempts int := 0;
BEGIN
  LOOP
    new_code := generate_referral_code();
    EXIT WHEN NOT EXISTS (SELECT 1 FROM referral_codes WHERE code = new_code);
    attempts := attempts + 1;
    IF attempts > 20 THEN
      RAISE EXCEPTION 'Could not generate unique referral code after 20 attempts';
    END IF;
  END LOOP;

  INSERT INTO referral_codes (user_id, code)
  VALUES (NEW.id, new_code)
  ON CONFLICT (user_id) DO NOTHING;

  -- Also bootstrap credit balance row
  INSERT INTO user_credit_balance (user_id)
  VALUES (NEW.id)
  ON CONFLICT (user_id) DO NOTHING;

  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

DROP TRIGGER IF EXISTS trg_create_referral_code ON users;
CREATE TRIGGER trg_create_referral_code
  AFTER INSERT ON users
  FOR EACH ROW EXECUTE FUNCTION trg_fn_create_referral_code();

-- ============================================================
-- BACKFILL: generate codes + balance rows for existing users
-- ============================================================
DO $$
DECLARE
  usr      RECORD;
  new_code text;
  chars    text := 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
  i        int;
BEGIN
  FOR usr IN
    SELECT id FROM users
    WHERE id NOT IN (SELECT user_id FROM referral_codes)
  LOOP
    LOOP
      new_code := 'DOF';
      FOR i IN 1..6 LOOP
        new_code := new_code || substr(chars, floor(random() * length(chars) + 1)::int, 1);
      END LOOP;
      EXIT WHEN NOT EXISTS (SELECT 1 FROM referral_codes WHERE code = new_code);
    END LOOP;
    INSERT INTO referral_codes (user_id, code) VALUES (usr.id, new_code)
    ON CONFLICT DO NOTHING;
  END LOOP;

  -- Bootstrap balance rows for existing users who don't have one
  INSERT INTO user_credit_balance (user_id)
  SELECT id FROM users
  WHERE id NOT IN (SELECT user_id FROM user_credit_balance)
  ON CONFLICT DO NOTHING;
END;
$$;

-- ============================================================
-- COLUMN: bookings.wallet_credits_applied_inr
-- Tracks how much credit wallet was used for a booking.
-- ============================================================
ALTER TABLE bookings
  ADD COLUMN IF NOT EXISTS wallet_credits_applied_inr int NOT NULL DEFAULT 0;

-- ============================================================
-- RLS POLICIES
-- ============================================================

-- referral_codes: users see only their own code
ALTER TABLE referral_codes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can read own referral code"
  ON referral_codes FOR SELECT
  USING (auth.uid() = user_id);

-- Public code lookup (needed for sign-up validation — anon can check if a code exists)
-- Note: only code text is exposed, not user_id
CREATE POLICY "Anyone can validate a referral code"
  ON referral_codes FOR SELECT
  USING (true);
-- Override: restrict to service role for write operations
CREATE POLICY "Service role full access to referral_codes"
  ON referral_codes FOR ALL
  TO service_role
  USING (true) WITH CHECK (true);

-- referral_redemptions: users see redemptions where they are referrer or referee
ALTER TABLE referral_redemptions ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can read own referral redemptions"
  ON referral_redemptions FOR SELECT
  USING (auth.uid() = referrer_user_id OR auth.uid() = referee_user_id);

CREATE POLICY "Service role full access to referral_redemptions"
  ON referral_redemptions FOR ALL
  TO service_role
  USING (true) WITH CHECK (true);

-- user_credit_balance: users see only their own balance
ALTER TABLE user_credit_balance ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can read own credit balance"
  ON user_credit_balance FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Service role full access to user_credit_balance"
  ON user_credit_balance FOR ALL
  TO service_role
  USING (true) WITH CHECK (true);

-- credit_wallet_transactions: users see own history
ALTER TABLE credit_wallet_transactions ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can read own credit transactions"
  ON credit_wallet_transactions FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Service role full access to credit_wallet_transactions"
  ON credit_wallet_transactions FOR ALL
  TO service_role
  USING (true) WITH CHECK (true);
```

```typescript
import type { SupabaseClient } from '@supabase/supabase-js';

export const REFERRAL_REWARD_INR = 500;

/**
 * Returns the referral code for a user, or null if not yet generated.
 */
export async function getReferralCode(supabase: SupabaseClient, userId: string) {
  const { data, error } = await supabase
    .from('referral_codes')
    .select('code, total_referrals, created_at')
    .eq('user_id', userId)
    .maybeSingle();
  if (error) throw error;
  return data;
}

/**
 * Looks up the referrer's user_id from a code string.
 * Returns null if code does not exist.
 */
export async function resolveReferralCode(supabase: SupabaseClient, code: string) {
  const normalised = code.trim().toUpperCase();
  const { data, error } = await supabase
    .from('referral_codes')
    .select('user_id, code')
    .eq('code', normalised)
    .maybeSingle();
  if (error) throw error;
  return data;
}

/**
 * Returns stats for a referrer: how many referrals they have made
 * and total credits earned from referrals.
 */
export async function getReferralStats(supabase: SupabaseClient, userId: string) {
  const { data, error } = await supabase
    .from('referral_redemptions')
    .select('id, status, referee_user_id, referrer_credited_at, created_at')
    .eq('referrer_user_id', userId)
    .order('created_at', { ascending: false });
  if (error) throw error;
  const credited = (data ?? []).filter((r) => r.status === 'credited');
  return {
    total: data?.length ?? 0,
    credited: credited.length,
    creditsEarned: credited.length * REFERRAL_REWARD_INR,
    redemptions: data ?? [],
  };
}

/**
 * Checks whether a referee has already used a referral code.
 * Returns true if the user already has a redemption row.
 */
export async function hasRedeemedReferral(supabase: SupabaseClient, refereeUserId: string) {
  const { data, error } = await supabase
    .from('referral_redemptions')
    .select('id')
    .eq('referee_user_id', refereeUserId)
    .maybeSingle();
  if (error) throw error;
  return data !== null;
}
```

```typescript
import type { SupabaseClient } from '@supabase/supabase-js';
import { createAdminClient } from '@/lib/supabase/admin-client';

/** Returns the current balance for a user. */
export async function getCreditBalance(supabase: SupabaseClient, userId: string) {
  const { data, error } = await supabase
    .from('user_credit_balance')
    .select('available_inr, lifetime_earned_inr, lifetime_used_inr, updated_at')
    .eq('user_id', userId)
    .maybeSingle();
  if (error) throw error;
  return data ?? { available_inr: 0, lifetime_earned_inr: 0, lifetime_used_inr: 0, updated_at: null };
}

/** Returns the last N credit transactions for a user. */
export async function getCreditHistory(supabase: SupabaseClient, userId: string, limit = 20) {
  const { data, error } = await supabase
    .from('credit_wallet_transactions')
    .select('id, amount_inr, transaction_type, reference_id, notes, balance_after, created_at')
    .eq('user_id', userId)
    .order('created_at', { ascending: false })
    .limit(limit);
  if (error) throw error;
  return data ?? [];
}

/**
 * Grants credits to a user.
 * Uses the service-role admin client to bypass RLS — must only be called from server-side code.
 * Returns the new balance.
 */
export async function grantCredits(
  userId: string,
  amountInr: number,
  transactionType: string,
  referenceId: string | null,
  notes: string,
): Promise<number> {
  const adminSupabase = createAdminClient();

  // Upsert balance (ensure row exists)
  const { data: balance, error: balError } = await adminSupabase
    .from('user_credit_balance')
    .upsert({ user_id: userId, available_inr: 0, lifetime_earned_inr: 0, lifetime_used_inr: 0 }, { onConflict: 'user_id', ignoreDuplicates: true })
    .select('available_inr, lifetime_earned_inr')
    .single();
  if (balError) throw balError;

  const newAvailable = (balance?.available_inr ?? 0) + amountInr;
  const newLifetime = (balance?.lifetime_earned_inr ?? 0) + amountInr;

  const { error: updateError } = await adminSupabase
    .from('user_credit_balance')
    .update({
      available_inr: newAvailable,
      lifetime_earned_inr: newLifetime,
      updated_at: new Date().toISOString(),
    })
    .eq('user_id', userId);
  if (updateError) throw updateError;

  const { error: txnError } = await adminSupabase.from('credit_wallet_transactions').insert({
    user_id: userId,
    amount_inr: amountInr,
    transaction_type: transactionType,
    reference_id: referenceId,
    notes,
    balance_after: newAvailable,
  });
  if (txnError) throw txnError;

  return newAvailable;
}

/**
 * Deducts credits from a user's wallet for a booking.
 * Throws if insufficient balance.
 * Must only be called from server-side code.
 */
export async function deductCredits(
  userId: string,
  amountInr: number,
  bookingId: string,
): Promise<number> {
  const adminSupabase = createAdminClient();

  const { data: balance, error: balError } = await adminSupabase
    .from('user_credit_balance')
    .select('available_inr, lifetime_used_inr')
    .eq('user_id', userId)
    .single();
  if (balError) throw balError;

  if ((balance?.available_inr ?? 0) < amountInr) {
    throw new Error('Insufficient credit balance.');
  }

  const newAvailable = balance.available_inr - amountInr;
  const newUsed = (balance.lifetime_used_inr ?? 0) + amountInr;

  const { error: updateError } = await adminSupabase
    .from('user_credit_balance')
    .update({
      available_inr: newAvailable,
      lifetime_used_inr: newUsed,
      updated_at: new Date().toISOString(),
    })
    .eq('user_id', userId);
  if (updateError) throw updateError;

  const { error: txnError } = await adminSupabase.from('credit_wallet_transactions').insert({
    user_id: userId,
    amount_inr: -amountInr,
    transaction_type: 'booking_applied',
    reference_id: bookingId,
    notes: `Applied to booking ${bookingId}`,
    balance_after: newAvailable,
  });
  if (txnError) throw txnError;

  return newAvailable;
}

/**
 * Restores credits to a user's wallet when a booking is cancelled.
 */
export async function restoreCredits(
  userId: string,
  amountInr: number,
  bookingId: string,
): Promise<number> {
  const adminSupabase = createAdminClient();

  const { data: balance, error: balError } = await adminSupabase
    .from('user_credit_balance')
    .select('available_inr, lifetime_used_inr')
    .eq('user_id', userId)
    .single();
  if (balError) throw balError;

  const newAvailable = (balance?.available_inr ?? 0) + amountInr;
  const newUsed = Math.max(0, (balance?.lifetime_used_inr ?? 0) - amountInr);

  const { error: updateError } = await adminSupabase
    .from('user_credit_balance')
    .update({
      available_inr: newAvailable,
      lifetime_used_inr: newUsed,
      updated_at: new Date().toISOString(),
    })
    .eq('user_id', userId);
  if (updateError) throw updateError;

  const { error: txnError } = await adminSupabase.from('credit_wallet_transactions').insert({
    user_id: userId,
    amount_inr: amountInr,
    transaction_type: 'booking_refunded',
    reference_id: bookingId,
    notes: `Refunded from cancelled booking ${bookingId}`,
    balance_after: newAvailable,
  });
  if (txnError) throw txnError;

  return newAvailable;
}
```

```typescript
// app/api/referrals/validate/route.ts
// Body: { code: string }
// Returns: { valid: boolean, message?: string }
export async function POST(request: Request) {
  // Rate-limited: 10 req/min per IP
  const { code } = await request.json();
  if (!code || typeof code !== 'string') {
    return NextResponse.json({ valid: false, message: 'No code provided.' });
  }
  const adminSupabase = createAdminClient();
  const resolved = await resolveReferralCode(adminSupabase, code);
  if (!resolved) {
    return NextResponse.json({ valid: false, message: 'Invalid referral code.' });
  }
  return NextResponse.json({ valid: true });
}
```

### `POST /api/referrals/redeem`

**Purpose**: Called by `complete-profile` internally (server-to-server, not client-facing). Creates the redemption record and grants credits to both parties atomically.

This is NOT a public route. It is called internally from `complete-profile/route.ts` after the user is created, passing the referral code.

```typescript
// Internal function in lib/referrals/service.ts (not a separate route)
export async function redeemReferralCode(
  refereeUserId: string,
  rawCode: string,
): Promise<{ success: boolean; message: string }> {
  const adminSupabase = createAdminClient();
  const code = rawCode.trim().toUpperCase();

  // 1. Resolve code → referrer
  const codeRow = await resolveReferralCode(adminSupabase, code);
  if (!codeRow) return { success: false, message: 'Invalid referral code.' };

  const referrerUserId = codeRow.user_id;

  // 2. Block self-referral
  if (referrerUserId === refereeUserId) {
    return { success: false, message: 'Cannot use own referral code.' };
  }

  // 3. Check referee has not already redeemed
  const alreadyRedeemed = await hasRedeemedReferral(adminSupabase, refereeUserId);
  if (alreadyRedeemed) return { success: false, message: 'Referral code already used.' };

  // 4. Create redemption record
  const { data: redemption, error: redemptionError } = await adminSupabase
    .from('referral_redemptions')
    .insert({
      referral_code: code,
      referrer_user_id: referrerUserId,
      referee_user_id: refereeUserId,
      status: 'pending',
    })
    .select('id')
    .single();
  if (redemptionError) throw redemptionError;

  // 5. Grant ₹500 to referee
  await grantCredits(
    refereeUserId,
    REFERRAL_REWARD_INR,
    'referral_reward_referee',
    redemption.id,
    `Welcome credit from referral code ${code}`,
  );

  // 6. Grant ₹500 to referrer
  await grantCredits(
    referrerUserId,
    REFERRAL_REWARD_INR,
    'referral_reward_referrer',
    redemption.id,
    `Referral reward — your friend joined Dofurs using your code`,
  );

  // 7. Update redemption to credited + increment referrer's count
  await adminSupabase
    .from('referral_redemptions')
    .update({
      status: 'credited',
      referrer_credited_at: new Date().toISOString(),
      referee_credited_at: new Date().toISOString(),
    })
    .eq('id', redemption.id);

  await adminSupabase
    .from('referral_codes')
    .update({ total_referrals: adminSupabase.rpc('increment', { x: 1 }) })
    .eq('user_id', referrerUserId);
  // Note: use a raw SQL increment or SELECT + UPDATE if rpc increment is not configured:
  // SELECT total_referrals FROM referral_codes WHERE user_id = X → update total_referrals + 1

  // 8. Send notifications to both
  await adminSupabase.from('notifications').insert([
    {
      user_id: refereeUserId,
      type: 'referral_welcome_credit',
      title: '₹500 Dofurs Credits Added!',
      body: `Welcome to Dofurs! ₹500 credits have been added to your wallet. Use them on any pet care booking.`,
      data: { amount_inr: REFERRAL_REWARD_INR },
    },
    {
      user_id: referrerUserId,
      type: 'referral_reward_credited',
      title: '₹500 Referral Reward Earned!',
      body: `Your referral just joined Dofurs! ₹500 credits have been added to your wallet.`,
      data: { amount_inr: REFERRAL_REWARD_INR, referral_code: code },
    },
  ]);

  return { success: true, message: 'Referral redeemed successfully.' };
}
```

## 16. Abuse Prevention

| Rule | Where enforced |
|---|---|
| One referral code per user at sign-up | `UNIQUE (referee_user_id)` on `referral_redemptions` |
| Cannot self-refer | `referrerUserId !== refereeUserId` check in `redeemReferralCode()` |
| Code must exist | DB lookup; returns 400 if not found |
| Code redemption non-fatal to sign-up | Try/catch in `complete-profile` |
| Rate limit on `/api/referrals/validate` | `isRateLimited()` — 10 req/min per IP |
| Credits deducted atomically | Read balance → check → update + insert txn in same service call |
| Booking cancellation restores credits | Call `restoreCredits()` in cancellation state transition |
